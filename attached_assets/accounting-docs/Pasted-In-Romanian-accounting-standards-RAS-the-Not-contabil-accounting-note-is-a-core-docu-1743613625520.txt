In **Romanian accounting standards (RAS)**, the **Notă contabilă** (accounting note) is a *core document* used in double-entry bookkeeping to record financial transactions in the general ledger. It is strictly regulated and plays a central role in ensuring auditability, traceability, and compliance with both **OMFP 1802/2014** and **OMFP 2844/2016** (for IFRS users).

---

## ✅ Key Elements of a Notă Contabilă

A valid *notă contabilă* must include:

| Element                      | Description                                                                 |
|-----------------------------|-----------------------------------------------------------------------------|
| **Număr (ID)**              | Unique, sequential identifier (internally generated in ERP)                |
| **Data întocmirii**         | The date the accounting entry is generated                                 |
| **Data operațiunii**        | The date of the actual transaction                                          |
| **Cont Debitor**            | Account to be debited (e.g., `4111` – Clienți)                              |
| **Cont Creditor**           | Account to be credited (e.g., `707` – Venituri din vânzarea mărfurilor`)   |
| **Sumă**                    | Amount of the transaction (in RON)                                          |
| **Valută (opțional)**       | Currency and exchange rate, if not in RON                                  |
| **Explicații (descriere)**  | Clear description of the transaction (ex: "Factura 12345 client X")        |
| **Tip document sursă**      | Invoice, contract, payroll statement, etc.                                 |
| **Număr document sursă**    | External/internal doc number                                               |
| **Departament / Centru cost** | Optional, for cost tracking                                               |
| **Responsabil întocmire**   | Name or signature of the responsible person (accountant/operator)          |

---

## 📘 Legal Context & Usage

- **Based on partida dublă (double-entry accounting)**.
- Regulated via **OMFP 1802/2014**, Art. 6 and Annex 1, plus:
  - **Legea contabilității 82/1991** (Art. 6-10)
- Not required for each invoice in detail, but **mandatory for aggregated journal entries**, end-of-month closings, provisions, adjustments, etc.
- Stored as part of **Registrul Jurnal** (Journal Ledger).
- Must be generated and archived **chronologically**, and **cannot be deleted** once the period is closed.

## 🧮 Example of a Simple Notă Contabilă

```
Data:         31.03.2025  
Număr:        NC-000234  
Explicații:   Înregistrare venituri din factură client ABC  
Document:     Factura F1234

Cont Debitor: 4111 - Clienți                         1.190,00 RON  
Cont Creditor: 707 - Venituri din vânzarea mărfii    1.000,00 RON  
Cont Creditor: 4427 - TVA colectată                   190,00 RON
```

## 📌 Best Practices in ERP

- Each ERP system must allow **manual and automatic generation** of note contabile.
- Every module (facturi, salarizare, gestiune, amortizări) must **emit automatic notă contabilă**.
- Link to source transaction (invoice, stock movement, etc.) for auditability.
- In GeniusERP: Store in a `journal_entries` table with `entry_lines`, using `account_id`, `debit`, `credit`, `source_doc`, etc.
